🚨 Problema Identificado
O campo priority funciona com valores em INGLÊS no banco de dados, mas há inconsistências no código:

Dados no banco: 'low', 'medium', 'high', 'critical' ✅
Labels de exibição: 'Baixa', 'Média', 'Alta', 'Crítica' ✅
Schema de validação: Correto (inglês) ✅
Status: Tem problema de mapeamento ❌
🔧 Por que Status tem problema e Priority não?
Nos logs você pode ver:

"❌ Form errors:",{"status":{"message":"Invalid enum value. Expected 'novo' | 'aberto' | 'em_andamento' | 'resolvido' | 'fechado', received 'in_progress'"}}
O problema é que:

Priority: Dados no banco (low, medium, etc.) coincidem com o schema de validação ✅
Status: Dados no banco (new, in_progress, etc.) NÃO coincidem com o schema que espera português ❌
📋 Resposta Final
O campo de prioridade trabalha com dados em INGLÊS (low, medium, high, critical) tanto no banco quanto no schema de validação, por isso funciona corretamente. O problema está apenas no campo status que tem divergência entre os valores armazenados (inglês) e o schema de validação (português).
